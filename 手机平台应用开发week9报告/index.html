    ###
  1. 如Figure 1至Figure 8所示，本次实验演示应用包含两个Activity。
  2. 首先是密码输入Activity：
    • 若应用首次启动，则界面呈现出两个输入框，分别为新密码输入框和确认密码输入框。
    • 输入框下方有两个按钮：
      • OK按钮点击后：
      • 若New Password为空，则发出Toast提示。见Figure 3。
      • 若New Password与Confirm Password不匹配，则发出Toast提示，见Figure 2。
      • 若两密码匹配，则保存此密码，并进入文件编辑Activity。
      • CLEAR按钮点击后：清楚两输入框的内容。
    • 完成创建密码后，退出应用再进入应用，则只呈现一个密码输入框，见Figure 4。
      • 点击OK按钮后，若输入的密码与之前的密码不匹配，则弹出Toast提示，见Figure 5。
      • 点击CLEAR按钮后，清除密码输入框的内容。
    • 出于演示和学习的目的，本次实验我们使用SharedPreferences来保存密码。但实际应用中不会使用这种方式来存储敏感信息，而是采用更安全的机制。见这里和这里。
  3. 文件编辑Activity：
    • 界面底部有三个按钮，高度一致，顶对齐，按钮水平均匀分布，三个按钮上方除ActionBar和StatusBar之外的全部空间由一个EditText占据（保留margin）。EditText内的文字竖直方向置顶，左对齐。
    • 在编辑区域输入任意内容，点击SAVE按钮后能保存到指定文件（文件名随意）。成功保存后，弹出Toast提示，见Figure 8。
    • 点击CLEAR按钮，能清空编辑区域的内容。
    • 点击LOAD按钮，能够从同一文件导入内容，并显示到编辑框中。若成功导入，则弹出Toast提示。见Figure 7.若读取文件过程中出现异常（如文件不存在），则弹出Toast提示。见Figure 6.
  4. 特殊要求：进入文件编辑Activity后，若点击返回按钮，则直接返回Home界面，不再返回密码输入Activity。

四、实验结果

1. 实验截图

(1)创建密码界面

应用第一次启动，进入创建密码界面，两个输入框，两个按钮。其中CLEAR按钮用来清除输入框内容，OK按钮进行创建密码。

(2)文本编辑界面

创建密码成功，进入文本编辑界面，有一个文本内容输入框，有三个按钮：OK、LOAD、CLEAR。

  • OK按钮将输入框的文本内容保存到文件中。
  • LOAD按钮读取文本内容导入到输入框中。(若没有文件，则读取失败)
  • CLEAR按钮清空输入框文本内容。

(2)重新进入应用界面

如果你已经创建好密码，重新进入应用是密码登陆界面，只有一个密码验证输入框，只有输入正确的密码才能进入文本编辑界面。

Android 属性设置android:noHistory=“true”

具体内容：

  • MainActivity.java处理按钮的简单响应

    • CLEAR按钮清空文本框内容
    • 当文本框为空，点击OK按钮的Toast错误提示
    • 当两个文本框输入不一致，点击OK按钮的Toast错误提示

  • MainActivity.java处理按钮保存密码信息的响应，利用SharedPreferences进行xml格式的读写。在Android Studio通过View->Tool Windows-> Device File Explorer中打开设备文件管理器，根据路径/data/data/<package name>/shared_prefs/查看保存的xml文件。

  • AndroidMainfest.xml中设置注册Activity的noHistory属性，实现当Activity不可见时，在activity stack中除去。（也即按返回键直接返回Home）

  • MainActivity.java处理OnCreate()函数，判断是否已经保存了密码信息，若还没保存，则进入新建密码界面，若已经保存了，进入密码登陆界面（也即相应修改布局显示）

关键代码：

MainActivity.java对于新建密码成功的响应处理：

// 注册成功！用SharedPreferences保存密码信息
Context context = MainActivity.this;
SharedPreferences sharedPref = context.getSharedPreferences("MY_PASSWORD", Context.MODE_PRIVATE);
SharedPreferences.Editor editor = sharedPref.edit();
editor.putString("password", editText_confirm_password.getText().toString());
editor.commit();
// 跳转到FileEditorActivity
Intent intent = new Intent(MainActivity.this, FileEditorActivity.class);
startActivity(intent);

MainActivity.java中OnCreate()函数进行对界面模式的判断：（注册or登陆）

// 确认显示模式：第一次加载为注册模式，保存密码之后就登陆模式
final boolean register_mode = false;
final boolean login_mode = true;
SharedPreferences sp = MainActivity.this.getSharedPreferences("MY_PASSWORD", Context.MODE_PRIVATE);
final String p = sp.getString("password", "");
final boolean page_mode = (p.equals("")) ? register_mode : login_mode;

// 如果是登陆模式，UI进行改变
if(page_mode == login_mode) {
    editText_new_password.setVisibility(View.INVISIBLE);
    editText_confirm_password.setHint("Password");
}

Android中byte数组与Sting的互相转换

具体内容：

  • FileEditorActivity.java实现按钮的简单响应
    • CLEAR按钮清空文本框
  • FileEditorActivity.java实现点击OK按钮后保存信息，利用FileOutputStream将文本框内容写入文件中。
  • FileEditorActivity.java实现LOAD按钮的保存信息，利用FileInputStream将读取文件内容，并在文本框显示。若文本不存在，则Toast提示错误。

关键代码：

  • 利用FileOutputStream将文本框内容写入文件：
// 将输入框的信息保存至文件中
try (FileOutputStream fileOutputStream = openFileOutput("FILE_CONTENT", Context.MODE_PRIVATE)) {
    String str = editText_file_content.getText().toString();
    fileOutputStream.write(str.getBytes());
    Toast.makeText(FileEditorActivity.this, "Save Successfully.", Toast.LENGTH_SHORT).show();
} catch (IOException ex) {
    Toast.makeText(FileEditorActivity.this, "Fail to save file.", Toast.LENGTH_SHORT).show();
}
  • 利用FileInputStream将文件文本内容显示在文本框：
// 将文件文本内容显示到文本框中
try (FileInputStream fileInputStream = openFileInput("FILE_CONTENT")) {
    byte[] contents = new byte[fileInputStream.available()];
    fileInputStream.read(contents);
    String str = new String(contents, "UTF-8");
    editText_file_content.setText(str);
    Toast.makeText(FileEditorActivity.this, "Load Successfully.", Toast.LENGTH_SHORT).show();
} catch (IOException ex) {
    Toast.makeText(FileEditorActivity.this, "Fail to load file.", Toast.LENGTH_SHORT).show();
}

3. 实验遇到的困难以及解决思路

1.editText多行输入置顶左对齐

解决方案：xml文件里面设置属性：android:gravity="top"

参考博客：android里面EditTex多行输入及输入置顶问题

2.跳转至FileEditorActivity闪退

解决方案：AndroidMainfest.xml中注册FileEditorActivity

3.当 Activity 不可⻅时，如何将其从 activity stack 中除去（按返回键直接返回Home）？

解决方案：AndroidManifest.xml 中设置 noHistory 属性。

参考博客： Android 属性设置android:noHistory=“true”

4.Android读取文件后得到的byte[]如何转换成String

解决方案：new String(byte_data, charsetName)

参考博客：Android中byte数组与Sting的互相转换
